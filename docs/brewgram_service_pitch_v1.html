B사용자 여정
07

사장님이 실제로 쓰는 순서

1

브랜드 온보딩

가게 이름, 색상, 분위기, 로고, 참고 인스타를 입력한다.

2

인스타 연결

전문 계정을 연결해 피드와 스토리 업로드 준비를 끝낸다.

3

상품 선택

신상품은 사진을 올리고, 기존 상품은 저장된 이미지를 재사용한다.

4

목적 선택

신메뉴, 시즌 한정, 이벤트, 영업 안내 등 게시 목적을 고른다.

5

광고 검수

문구와 이미지를 확인하고 필요한 경우 다시 생성한다.

6

업로드와 축적

피드·스토리에 올리고 결과물은 다음 생성의 히스토리로 남긴다.

차별점 기존 상품은 매번 다시 찍거나 설명하지 않아도 된다. 브랜드와 상품 자산이 쌓일수록 사용 부담이 줄어든다.
B데모 시나리오
08

처음 보는 사람도 이해하기 쉬운 데모 상황으로 보여준다

딸기 티라미수 신메뉴 홍보
  • 사장님은 오늘 신메뉴를 인스타그램에 바로 올려야 한다.
  • 상품 사진, 상품명, 목적처럼 이미 알고 있는 정보만 넣는다.
  • 온보딩에서 저장한 가게 분위기와 톤이 자동 반영된다.
  • 피드 문구, 상세 설명, 스토리 훅, 광고 이미지가 함께 나온다.
  • 결과를 검수한 뒤 인스타그램 업로드까지 이어진다.

B내부 측정 지표
10

시간 외 기대효과는 내부 로그로 확인한다

서비스 내부에서 측정 가능한 지표

  • 광고 생성 시작부터 업로드 요청 완료까지 걸린 시간
  • 생성 대비 업로드 요청 완료율
  • 기존 상품 재사용 비율
  • 생성 실패율과 재시도율
  • 브랜드 온보딩 후 반복 생성 횟수

사용자에게 설명할 효과

  • 카피 작성, 이미지 제작, 업로드 도구 전환을 한 흐름으로 축소
  • 브랜드 가이드가 매 생성마다 자동 반영되어 톤 일관성 유지
  • 신상품은 빠르게 등록하고 기존 상품은 다시 설명하지 않음
  • 피드와 스토리를 모두 고려해 모바일 홍보 채널 확장
측정 계획 다음 단계는 데모 시나리오에서 생성 시작, 결과 저장, 업로드 요청 완료까지의 로그를 남기는 것이다.
B서비스 포지셔닝
11

Brewgram은 생성 도구와 예약 도구 사이의 빈틈을 잡는다

범용 AI 도구

  • 프롬프트를 매번 새로 써야 한다.
  • 브랜드 톤이 이어지기 어렵다.
  • 업로드는 사용자가 따로 해야 한다.

SNS 예약·자동화 도구

  • 게시 일정과 운영 보조에 강점이 있다.
  • 콘텐츠 제작 흐름은 서비스마다 다르다.
  • 상품 사진과 목적 기반 생성은 별도 확인이 필요하다.

Brewgram

  • 브랜드를 기억하고 상품 광고를 만든다.
  • 피드와 스토리 결과물을 함께 제공한다.
  • 모바일에서 업로드까지 끝낸다.
포지셔닝 Brewgram은 이미지 하나를 만드는 도구가 아니라, 작은 가게의 인스타 홍보물을 끝까지 만드는 운영 흐름이다.

결론: 작은 가게의 반복 인스타 홍보 업무를 줄이는 서비스

핵심 메시지

  • 시장에는 홍보가 필요한 소상공인이 많다.
  • 고객 접점은 모바일과 인스타그램으로 이동했다.
  • 콘텐츠 제작은 꾸준히 하기 어렵고 시간이 든다.
  • Brewgram은 브랜드 기억, 광고 생성, 업로드를 하나로 묶는다.

다음 내부 보완 과제

  • 업로드 요청 성공·실패 로그 확인
  • 재생성률과 기존 상품 재사용률 추적
