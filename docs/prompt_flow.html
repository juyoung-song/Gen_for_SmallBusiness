🔀 프롬프트 흐름 — Gen_for_SmallBusiness

LLM 에 들어가는 모든 프롬프트의 재료 → 조립 → 모델 → 산출물 경로를 한눈에. 브랜치 refactor/flow 기준.

사용자 입력 / 외부 이미지
DB 저장 (brands·reference_images·generations 등)
프롬프트 빌더 함수
LLM 호출 (Langfuse observation)
산출물 / 외부 API

6. 설계 원칙 ↔ 프롬프트 방어선

원칙어디서 방어되는가
스타일 ≠ 구도brands.style_prompt vs reference_images.composition_prompt 물리 분리 + 각자의 system prompt 가 서로 반대 제약
참조는 톤 바꾸는 용도 아님build_composition_analysis_prompt의 STRICT RULES — 색/무드 단어 절대 금지
캡션에 구도 섞지 않음reference_analysis="" 하드코딩 (caption 호출부)
텍스트에 구도 섞지 않음결합 생성에서 req_t.reference_analysis="" (req_i 만 주입)
브랜드 불변brands 에 updated_at 없음, 수정 API 미노출
1계정 1브랜드UNIQUE(brands.instagram_account_id)
프롬프트→결과 추적Langfuse trace_id + generations.langfuse_trace_id
