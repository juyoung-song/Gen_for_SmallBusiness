# 📄 생성형 AI 기반 광고 콘텐츠 제작 서비스

## 최종 개발 기획서 (Execution PRD)

---

# 1. 프로젝트 개요

## 1.1 프로젝트 목적

본 프로젝트는 생성형 AI를 활용하여

**소상공인이 광고 콘텐츠를 쉽고 빠르게 제작할 수 있도록 지원하는 웹 서비스**를 개발하는 것을 목표로 한다.

사용자는 상품 정보와 간단한 요청을 입력하면

AI가 광고 문구 및 광고 이미지를 자동 생성한다.

---

## 1.2 개발 목표

본 프로젝트의 핵심 목표는 다음과 같다:

- 광고 문구 생성 기능 구현
- 광고 이미지 생성 기능 구현
- 사용자 입력 기반 콘텐츠 생성 서비스 구현
- 인스타그램 비즈니스 계정 자동 업로드 연동
- Streamlit 기반 MVP 웹 서비스 완성

---

## 1.3 개발 원칙

- 1인 개발 기준으로 **단순하고 유지보수 가능한 구조 유지**
- 모델 연구가 아닌 **서비스 완성도 중심 개발**
- 빠른 구현 → 점진적 개선 (Iterative Development)
- Mock → 실제 API 연동 순서로 개발

# 3. 기능 요구사항 (Functional Requirements)

---

## 3.1 광고 문구 및 전략 생성

### 입력

- 상품명 (필수)
- 상품 설명 (선택)
- **홍보 목적 (신상품 홍보, 할인 행사, 매장 소개, 시즌 홍보 등 - 필수)**
- 광고 스타일 (선택)
- 업로드 이미지 (선택 - 무드 및 특징 참고)

### 출력

- [광고 문구] 피드용 한 줄 카피 3개
- [홍보 문장] SNS용 2~3문장 상세 설명 2개 (전략적 목적 반영)
- [스토리 카피] 인스타 스토리용 초단문 Hook 3개

### 요구사항

- **홍보 목적(Goal) 최우선 반영**: 선택한 목적에 따라 설득 논리와 어조가 달라짐
- 스타일 및 이미지 무드 반영 가능
- 재생성 가능

---

## 3.2 광고 비주얼 생성

### 입력

- 상품 정보 및 목적 (자동 전달)
- 이미지 스타일 옵션
- **업로드 이미지 (선택 - 구도 및 색감 레퍼런스로 활용)**

### 출력

- 광고 이미지 (피드용 1:1)
- **[추가] 스토리용 합성 이미지 (9:16 - 텍스트 오버레이 포함)**

### 요구사항

- **전략적 비주얼 연출**: 홍보 목적에 맞는 조명, 구도, 소품 배치 가이드 적용
- 업로드 이미지 레퍼런스 준수: 원본 상품의 정체성을 보존하며 광고급 품질로 생성
- 텍스트 미포함(피드용) 및 텍스트 포함(스토리용) 대응

---

## 3.5 [NEW] 인스타그램 스토리 생성 및 업로드

### 요구사항
- **세로형 레이아웃(9:16)**: 모바일 피드에 최적화된 세로형 이미지 생성 및 합성
- **Pillow 기반 텍스트 합성**: 생성된 스토리 카피 중 하나를 선택하여 이미지 상단/중단에 가독성 높게 합성
- **스토리 전용 업로드**: Meta Graph API의 `share_to_story` 규격을 준수하여 즉시 게시

---

## 3.3 사용자 인터페이스 (Streamlit)

### 입력 영역

- 상품명 입력
- 상품 설명 입력
- 스타일 선택
- 생성 타입 선택 (문구 / 이미지)

### 출력 영역

- 생성된 광고 문구
- 생성된 이미지

### UX 요구사항

- 한 페이지 내 멀티 탭 구성 (✨ 새로 만들기 / 🗄️ 히스토리 아카이브)
- 버튼 클릭 → 결과 출력
- 로딩 상태 표시
- 에러 처리

---

## 3.4 히스토리 아카이브 (데이터베이스)

### 요구사항
- **비동기 DB 저장**: 생성 완료된 문구 및 이미지 내역을 백그라운드에서 SQLite에 저장
- **내역 조회**: '히스토리 아카이브' 탭에서 과거 생성된 전체 내역을 시간 역순으로 조회 가능
- **상세 내역 보관**: 생성 당시의 상품명, 스타일, 프롬프트, 결과물(텍스트 및 이미지 파일 경로) 보존

# 5. 시스템 설계

## 5.1 전체 아키텍처

```
User
 ↓
Streamlit UI
 ↓
Controller (App Logic)
 ↓
Service Layer
  ├── Text Generation Service
  ├── Image Generation Service
  ├── Caption Service
  └── Instagram Upload Service ← [NEW]
 ↓
┌───────────────────────────────────────┐
│  External APIs                        │
│  · OpenAI GPT (텍스트/번역)            │
│  · Hugging Face Inference (이미지)     │
│  · FreeImage.host (이미지 호스팅)       │
│  · Meta Graph API (인스타 업로드)       │
└───────────────────────────────────────┘
 ↓
Result (UI 출력 + 인스타그램 피드 게시)
```

# 7. 데이터 흐름 정의

```
[콘텐츠 생성]
입력 → 전처리 → 프롬프트 생성 → AI 모델 호출 → 결과 반환 → UI 출력 → DB 저장

[인스타그램 업로드]
image_bytes → JPEG 변환 → FreeImage 호스팅(퍼블릭 URL) → Meta /media(컨테이너) → Meta /media_publish → 피드 게시
```
