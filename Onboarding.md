# 🚀 Brand Onboarding Process Guide

이 문서는 **'소상공인을 위한 맞춤형 AI 광고 서비스'**의 핵심 엔진인 브랜드 온보딩 프로세스의 동작 원리와 코드 흐름을 상세히 설명합니다.

---

## 1. 개요 (Overview)
온보딩은 단순한 회원가입 절차가 아닙니다. 사용자의 브랜드가 가진 **고유한 시각적·언어적 정체성(Visual & Verbal Identity)**을 AI가 학습하도록 하여, 생성되는 모든 콘텐츠에 일관성을 부여하는 **'골든 프롬프트(Global Style Prompt)'**를 생성하는 과정입니다.

---

## 2. 상세 동작 프로세스

### Phase 1: 상태 진단 및 진입 (`app.py`)
- **트리거**: 앱 실행 시 `BrandService.get_brand_config()`를 호출합니다.
- **로직**: SQLite DB의 `brand_configs` 테이블이 비어있으면 온보딩 화면을, 데이터가 있으면 메인 화면을 렌더링합니다.
- **초기화**: 사용자가 브랜드를 바꾸고 싶어할 경우, 해당 테이블을 비우면 다시 온보딩 단계로 진입합니다.

### Phase 2: 데이터 수집 (Data Collection)
Streamlit UI를 통해 사장님으로부터 다음 정보를 수집합니다.
1. **브랜드 기본 정보**: 이름, 선호 분위기(Atmosphere), 브랜드 컬러(Hex).
2. **스타일 가이드라인**: 사용자가 직접 정의하는 브랜드의 지향점.
3. **핵심 레퍼런스**: 
   - **Reference Image**: 인스타그램 피드 캡처 등 사장님이 추구하는 구체적인 이미지 샘플 (바이너리 데이터).
   - **Reference URL**: 참고할 사이트 링크 (텍스트 데이터).

### Phase 3: AI 브랜드 해부 및 분석 (`AnalysisService`)
이 단계가 서비스의 핵심 기술입니다. **GPT-4o Vision** 모델을 활용합니다.
- **시각 분석 (Visual DNA Extraction)**:
  - 업로드된 이미지를 사진학적으로 분석합니다.
  - **추출 항목**: 렌즈 특성(85mm, f1.8), 조명 값(Side lighting, High-key), 색보정(Film grain, Desaturated), 구도(Centered hero shot).
- **프롬프트 엔지니어링**:
  - 수집된 텍스트와 시각 분석 결과를 결합하여 **영어 기반의 전역 스타일 프롬프트(Global Style Prompt)**를 생성합니다.
  - 결과물은 약 60~80단어의 전문적인 사진 지시어로 구성됩니다.

### Phase 4: 영구 저장 (Persistence)
- 분석된 결과는 `BrandService`를 통해 SQLite(`data/history.db`)에 저장됩니다.
- **저장 필드**: `brand_name`, `atmosphere`, `brand_color`, `global_style_prompt` 등.

---

## 3. 온보딩 결과의 활용 (Application)

온보딩 당시에 생성된 `style_prompt`는 사장님이 광고를 제작할 때마다 모든 명령어의 **최우선 순위**로 주입됩니다.

```python
# utils/prompt_builder.py (실제 주입 로직)
"STRICT BRAND VISUAL IDENTITY: [온보딩 시 분석된 스타일 프롬프트]"
+ "[현재 만들려는 상품 정보]"
+ "[홍보 목적 및 연출 의도]"
```

이 구조 덕분에 사장님은 매번 스타일을 설명할 필요 없이, 상품명만 입력해도 항상 **처음 설정한 그 브랜드 고유의 느낌**이 담긴 고품질 광고 사진을 얻게 됩니다.

---

## 5. 현재 구현 상태 및 기술적 한계 (Current Status & Technical Gaps)

시스템 분석 결과, 현재 기획된 내용과 실제 코드 구현 사이의 간극은 다음과 같습니다.

### ✅ 구현 완료 (DONE)
- **온보딩 UI 및 데이터 파이프라인**: 브랜드 정보를 수집하고 DB에 영구 저장하는 전체 흐름.
- **정밀 시각 분석 엔진**: GPT-4o Vision을 활용한 레퍼런스 이미지의 사진학적 특징(조명, 구도, 렌즈값 등) 추출 성공.
- **브랜드 스타일 강제 주입 로직**: 이미지 생성 시 브랜드 고유 스타일을 최우선 순위(Prefix)로 배치하여 일관성을 확보하는 명령어 체계.

### ⚠️ 미비 및 개선 과제 (TO-DO & GAPS)
1. **URL 기반 크롤링 부재**:
   - **상태**: 현재 Reference URL은 단순 텍스트로만 취급됨.
   - **문제**: 사용자가 링크만 넣을 경우 실제 이미지를 보지 못해 스타일 분석이 불가능함.
   - **해결**: 크롤링 모듈 추가 또는 '이미지 업로드 필수' UX 가이드 강화 필요.
2. **인스타그램 자동 업로드의 실효성**:
   - **상태**: 로직상 모의(Mock) 기능으로 구현되어 있을 가능성 높음.
   - **문제**: 실제 Facebook API 연동을 위한 보안 토큰 처리 및 승인 절차가 복잡함.
   - **해결**: 실제 API 연동 또는 단계별 가이드 제공 필요.
3. **로컬 모델 최적화**:
   - **상태**: 사용자의 로컬 환경(예: Mac)에서 고성능 AI(IP-Adapter) 가동 시 부하 발생.
   - **해결**: 저토큰 고효율 모델(FLUX.1-schnell 등 API 방식)로의 전환 또는 로컬 가속 최적화 필요.

---

## 6. 향후 발전 방향 (Roadmap)
- **정확도 향상**: 시각 분석 지시어의 세분화로 브랜드 필터 느낌 재현율 극대화.
- **UX 개선**: 레퍼런스 링크 입력 시 "이미지 캡처 업로드 권장" 안내 메시지 추가.
- **자동화 확장**: 실제 인스타그램 비즈니스 API와의 연동을 통한 완전 자동 게시 시스템 구축.
